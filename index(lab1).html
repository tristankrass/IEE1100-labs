Küsimus 1. GAUGE: Mida signaalitugevuse näit näitab, kuidas seda mõõdetakse ja milleks seda tulemust mobiilsides kasutatakse? Signaalitugevuse näit näitab, kui hea ühendus on telefoni ja tugijaama vahel (kui hea levi sul on). Signaalitugevust mõõdetakse tugijaamades dBm-ides. Tulemust kasutatakse selleks, et kasutajal oleks alati võimalikult hea levi (et tal oleks parim signaal) ehk kui mõnes tugijaamas on parem signaal, siis on kasutaja telefonil alati võimalus ennast sinna ümber lülitada.

Küsimus 2. GAUGE: Millist signaalitugevuse väärtuste vahemikku peetakse heaks, millist rahuldavaks ja millest allapoole ei mõõdeta? 2G (EDGE): >= -90 dBm - väga hea, -103 dBm kuni -90 dBm - hea, -107 dBm kuni -103 dBm - halb, <= -108 dBm ei mõõdeta, 3G (HSPA+): >= -90 dBm - väga hea, -104 dBm kuni -90 dBm- hea, -107 dBm kuni -104 dBm - halb, <= -108 dBm ei mõõdeta, 4G (LTE): >= -98 dBm - väga hea, -118 dBm kuni -98 dBm - hea, -128 dBm kuni -118 dBm - halb, <= -129 dBm ei mõõdeta

Küsimus 3. GAUGE: Millist informatsiooni saab teada sellest mõõteaknast veel (ühe-kahe lausega)? Mõõteaknast saab veel infot mobiilside teenusepakkuja kohta, võrgutüübi kohta, ühendust tagava tugijaama kohta, kõige tugevama signaaliga tugijaama signaalitugevuse kohta ja teenuse oleku kohta (S- Service Status, D- Mobile Data, R- Roaming).

2.3 RAW

Tehnoloogia Kuvatõmmis tehnoloogiast
4G 3G 2G

Küsimus 1. RAW: Mitut naabertugijaama mõõdetakse? 3 (2G ja 3G). 2 (4G)

Küsimus 2. RAW: Millist tehnilist infot kogutakse teenindava tugijaama kohta, millist naabertugijaamade kohta? Leida võimalikud erinevused kasutatud mobiilside tehnoloogiate vahel.

Nii teenindava tugijaama kui ka naabertugijaamade kohta kogutakse infot tugijaama identifitseeritavate koodide, sagedusala ning signaali tugevuse ja kvaliteedi info kohta. Teenindava tugijaama kohta kogutakse rohkem identifitseeritavaid koode ja signaali tugevuse ning kvaliteedi infot kui naabertugijaamade kohta. Mobiilside tehnoloogiate vahel on sarnasteks ainult riik (mcc) ja operaator(mnc). Kõik teised andmed on erinevad.

Küsimus 3. RAW: Selgitada välja, mida iga kuvatud arv tähendab, esitada selgitused aruandes.

- MCC - Mobile country code

- MNC - Mobile network code

- Band - The frequency which the given protocol works on.

- TAC - Tracking Area Code.

- LAC - location area code

- CID/UCID/ECI - Cell id 2G/3G/4G

- PSC - Primary Scrambling Code

- PCI - Physical Cell Identity

- eNB - eNodeB known as E-UTRAN Node B or Evolved Node B.

- RNC - Radio Network Controller

- RSSI - Received Signal Strength

- RSRP - Reference Signal Received Power

- ASU - Arbitrary Strength Unit

- Power - Signal strength in watts

- RSRQ - Reference Signal Received Quality

- RSSNR - Reference Signal Signal Noise Ratio

- TA- Timing Advance

Kokkuvõte ja järeldused

Tegu oli viimase laboriga arvutivõrkude aines ning labori lõpuks oli meel häre, sest rohkem laboreid ei tulegi. Labori raames sain kargel pühapäeva varahommikul ratta selga hüpata ja kolm ringi ümber tugijaamade sõita. Kindlasti andis labor palju juurde, just selle osas, et sain näha, kuidas reaalne sidepidamine käib tugijaama ja telefoni vahel. Midagi mida igapäevaselt võtame kui enesest mõistevana.
